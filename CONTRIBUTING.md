# Contributing

Thanks for improving CloudSync SafeOps. This repository is public-facing and sanitized, so contributions should be useful for a portfolio or demo without exposing real infrastructure.

## Contribution checklist

- Use placeholders only: `example.com`, `safeops.example.com`, `doh.example.com`, `monitor.example.com`, `203.0.113.10`, or `REDACTED`.
- Keep examples read-only and non-destructive.
- Do not add real logs, private configuration, credentials, backup files, database exports, or personal data.
- Keep language professional and operations-focused.
- Avoid sales-oriented or abuse-oriented positioning. The project is about a cybersecurity and DevOps learning lab.

## Documentation style

- Write for small business owners and technical reviewers.
- Prefer clear checklists and short runbooks.
- State limitations honestly.
- Do not promise perfect protection or uninterrupted availability.

## Before opening a pull request

Run the local checks where available:

```bash
bash scripts/healthcheck.example.sh
```

Then review the changed files manually for placeholders and clarity.
