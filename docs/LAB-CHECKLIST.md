# Lab Checklist

This checklist describes how to build a clean baseline for a generic infrastructure operations lab. It uses placeholders only and avoids live production values.

## Baseline Process

1. List the demo domain and service targets.
2. Define monitoring checks.
3. Confirm alert-routing placeholders.
4. Check SSL/TLS certificate status.
5. Check DNS records and DNS security assumptions.
6. Create a baseline report.
7. Create the backup readiness checklist.
8. Record the documentation update.

## Required Inputs

- Project name.
- Placeholder domains and service URLs.
- Critical hours for the scenario.
- Alert channel placeholder.
- Hosting or platform description.
- Backup readiness status.
- Known lab assumptions.

## Deliverables

- Monitoring target list.
- Baseline health report.
- Backup readiness checklist.
- Incident communication path.
- Monthly report template.
