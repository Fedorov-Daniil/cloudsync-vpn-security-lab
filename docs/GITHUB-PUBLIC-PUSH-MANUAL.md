# Public Repository Push Manual

If no safe remote exists:

1. Create a PUBLIC GitHub repository named `cloudsync-vpn-security-lab`.
2. Description: `Sanitized VPN security operations lab: DNS protection, monitoring, backups, rollback planning, and safe production change practices.`
3. Suggested topics: `linux`, `docker`, `dns-security`, `devops`, `monitoring`, `backups`, `cybersecurity`, `homelab`, `adguard`, `doh`.
4. Add a remote without embedding credentials in the URL.
5. Push:

   git push -u origin main

6. Review the repository in the browser and confirm it contains no sensitive operational values.
